Budget Manager HTML Guide
=======================

This guide explains the HTML structure and components used in the Budget Manager application. It's written for developers who want to understand how the interface is built.

1. Basic Structure
----------------
```html
<!DOCTYPE html>
<html lang="en">
<head>
    <meta charset="UTF-8">
    <meta name="viewport" content="width=device-width, initial-scale=1.0">
    <title>Family Dashboard | Budget Manager</title>
    <!-- CSS Files -->
    <link rel="stylesheet" href="/src/assets/css/styles.css">
    <link rel="stylesheet" href="/src/assets/css/dashboard.css">
    <link rel="stylesheet" href="/src/assets/css/family.css">

    <link rel="stylesheet" href="https://cdnjs.cloudflare.com/ajax/libs/font-awesome/6.0.0/css/all.min.css">
</head>
```

9. Important Elements and IDs
--------------------------
Key elements that JavaScript interacts with:

1. Dashboard Elements:
   - userGreeting: Displays user's name
   - totalIncome: Shows total income
   - totalExpenses: Shows total expenses
   - balance: Shows current balance
   - familyMembers: Container for member cards
   - recentTransactions: Container for transactions
   - expenseChart: Canvas for expense chart

2. Modal Elements:
   - memberModal: Add member modal
   - transactionModal: Add transaction modal
   - memberForm: Form for adding members
   - transactionForm: Form for adding transactions

3. Action Buttons:
   - addMemberBtn: Opens member modal
   - addIncome: Opens transaction modal for income
   - addExpense: Opens transaction modal for expense
   - cancelMember: Closes member modal
   - cancelTransaction: Closes transaction modal

10. Best Practices
---------------
1. Semantic HTML:
   - Use appropriate tags (header, section, main)
   - Proper heading hierarchy (h1, h2, h3)
   - ARIA labels for better accessibility

2. Form Validation:
   - Required fields marked with required attribute
   - Appropriate input types (text, number)
   - Clear labels and placeholders

3. Modal Structure:
   - Proper focus management
   - Keyboard navigation support
   - Clear action buttons

4. Component Organization:
   - Logical grouping of related elements
   - Clear parent-child relationships
   - Consistent class naming

11. Accessibility Features
-----------------------
1. ARIA Roles:
   ```html
   <button aria-label="Delete member">
       <i class="fas fa-trash"></i>
   </button>
   ```

2. Focus Management:
   ```html
   <div class="modal" role="dialog" aria-labelledby="modalTitle">
       <h2 id="modalTitle">Add Member</h2>
   </div>
   ```

3. Form Labels:
   ```html
   <label for="memberName">Name</label>
   <input id="memberName" name="memberName" type="text"
          aria-required="true" aria-invalid="false">
   ```

Remember:
--------
1. Keep HTML structure semantic and meaningful
2. Use appropriate ARIA attributes for accessibility
3. Maintain consistent naming conventions
4. Ensure proper form validation attributes
5. Keep modals and dialogs accessible

13. Enhanced Accessibility Guidelines
--------------------------------
1. Keyboard Navigation:
```html
<!-- Skip Navigation -->
<a href="#main-content" class="skip-link">Skip to main content</a>

<!-- Keyboard-friendly buttons -->
<button class="action-button" 
        tabindex="0" 
        role="button"
        onkeypress="handleKeyPress(event)">
    Action
</button>
```

2. Screen Reader Considerations:
```html
<!-- Status Updates -->
<div role="status" aria-live="polite" class="sr-only">
    Transaction added successfully
</div>

<!-- Progress Indicators -->
<div class="progress-bar" 
     role="progressbar" 
     aria-valuenow="75" 
     aria-valuemin="0" 
     aria-valuemax="100">
    75%
</div>
```

3. Form Error Handling:
```html
<div class="form-group">
    <label for="amount">Amount</label>
    <input type="number" 
           id="amount" 
           aria-invalid="true"
           aria-errormessage="amountError">
    <div id="amountError" class="error-message" role="alert">
        Please enter a valid amount
    </div>
</div>
```

14. Troubleshooting Guide
----------------------
1. Common Issues and Solutions:

   a. Modal Dialog Issues:
   ```html
   <!-- Problem: Modal not trapping focus -->
   <!-- Solution: Add proper attributes -->
   <div class="modal" 
        role="dialog"
        aria-modal="true"
        aria-labelledby="modalTitle"
        tabindex="-1">
   ```

   b. Form Validation Problems:
   ```html
   <!-- Problem: Form submitting invalid data -->
   <!-- Solution: Add proper validation -->
   <form id="transactionForm" 
         novalidate
         data-validate="true"
         onsubmit="return validateForm(event)">
   ```

   c. Responsive Layout Issues:
   ```html
   <!-- Problem: Content overflow on mobile -->
   <!-- Solution: Add proper viewport meta -->
   <meta name="viewport" 
         content="width=device-width, 
                  initial-scale=1.0, 
                  maximum-scale=5.0">
   ```

2. Debug Checklist:
   - Validate HTML using W3C Validator
   - Check for duplicate IDs
   - Verify ARIA attributes
   - Test keyboard navigation
   - Confirm proper form validation
   - Check responsive breakpoints

```html

      as="font"
      type="font/woff2"
      crossorigin>
```
